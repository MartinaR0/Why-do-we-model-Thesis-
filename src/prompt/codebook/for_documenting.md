# For Documenting

## Definition
A goal of using a model to support asynchronous communication between people
about a domain of interest: documentation, knowledge base, or reference for
people not part of the original modeling process.

## Signal patterns
- Positive: documentation, knowledge base, reference, for future use,
  knowledge management, stored, archived, repository
- Negative: real-time, meeting, live discussion

(Keyword alone is not sufficient; the surrounding sentence must show the model
was kept or maintained as a reference for people outside the original modeling
process.)

## Disambiguation
**vs. For Communicating:** The distinction is temporal. Asynchronous/persistent
use (consulted later by absent parties) -> Documenting. Synchronous/live use
(meetings, real-time explanation) -> Communicating. If both clearly occur, code
both.

## Confidence
Present only at High or Medium.
