# For Intervening

## Definition
A goal of using a model as an instrument through which one changes something in
the world. The model directly guides or enables the creation of a system,
artifact, or implementation. The model is actively used to build something, not
merely proposed. Actual implementation is strong evidence, but an explicitly
stated intended use for implementation, integration, transformation, artifact
generation, or system development may also count when the paper frames this as
the model's purpose.

## Signal patterns
- Positive: implemented, deployed, built, used to create, used to enable
  interoperability, generated artifact
- Negative: would be implemented, future work, proposed only

(Keyword alone is not sufficient; the surrounding sentence must show the model
was actually used to build or change something.)

## Disambiguation
**vs. For Problem-solving:** If the model is changed and that change is reflected
in a schema, system, artifact, or implementation -> Intervening. If the model is
only used to identify shortcomings/problems/improvements -> Problem-solving. If
the same evidence only shows implementation, code Intervening, not
Problem-solving. Code both only when separate evidence shows decision-making or
analysis in addition to implementation.

## Confidence
Present only at High or Medium. Do not code future/hypothetical implementation as
present unless the paper frames it as the model's intended purpose.
