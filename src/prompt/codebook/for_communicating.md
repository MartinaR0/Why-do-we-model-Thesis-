# For Communicating

## Definition
A goal of using a model to support communication between people about a domain of
interest (meetings, explanations, live discussion bridging different groups). If
the paper states the model supports communication between stakeholders but does
not specify synchronous or asynchronous, code Communicating only when the context
suggests interaction, explanation, alignment, or discussion between distinct
people or groups.

## Signal patterns
- Positive: communicate, meeting, align stakeholders, bridge between, explain to,
  discussion
- Negative: stored for future, documentation system

## Disambiguation
**vs. For Documenting:** The distinction is temporal. Synchronous/live use
(meetings, real-time explanation) -> Communicating. Asynchronous/persistent use
(reference consumed later by absent parties) -> Documenting. If both clearly
occur, code both.

**vs. For Understanding:** Communicating requires conveying meaning between
distinct parties; internal sense-making by one person/team -> Understanding.

## Confidence
Present only at High or Medium.
